# README 업데이트 규칙

## 원칙

기능이 추가·변경·제거되면 `README.md`와 `README_kr.md`가 현재 상태와 일치하는지 확인하고 필요 시 함께 업데이트한다.

## 적용 시점

아래 작업이 있을 때마다 README 반영 여부를 검토한다.

- 도구(tool) 추가·삭제·이름 변경
- 도구 인자(특히 필수 인자)나 동작 변경
- 환경변수 추가·삭제·이름 변경
- 실행 방법(`npx`, 빌드 명령, 바이너리 경로 등) 변경
- 캐시 위치·포맷·초기화 방법 변경
- 안전장치(허용 SQL, 에러 복구 등) 동작 변경
- 라이선스, Node 요구 버전 등 메타정보 변경

## 절차

1. 변경 사항이 README에 언급된 영역인지 확인한다 (도구 표, 환경변수 표, 사용 예시, 안전장치, 데이터소스 규칙, 캐시 섹션 등)
2. 영향이 있다면 `README.md`와 `README_kr.md`를 **동시에** 수정한다 (둘 중 하나만 업데이트 금지)
3. 두 문서의 내용·구조·섹션 순서를 일치시킨다. 한 쪽에만 있는 정보가 없도록 한다
4. 코드 변경 커밋과 같은 커밋 또는 바로 이어지는 커밋에 README 업데이트를 포함한다

## 체크리스트

기능 변경 작업을 마무리하기 전에 확인:

- [ ] 도구 표의 목록/설명이 현재 구현과 일치하는가
- [ ] 환경변수 표가 실제 읽는 변수와 일치하는가
- [ ] 사용 예시의 호출 순서가 여전히 유효한가
- [ ] 안전장치·데이터소스·캐시 섹션의 설명이 코드와 일치하는가
- [ ] 영문/한글 두 파일이 같은 정보를 담고 있는가
