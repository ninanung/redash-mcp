# 파일 분리 규칙

## 원칙

하나의 파일에 코드가 지나치게 늘어나면 역할 단위로 별도 파일로 분리한다.

## 기준

- 한 파일이 200줄을 넘으면 분리를 검토한다
- 핸들러, 유틸리티, 상수 등 역할이 다른 코드가 섞여 있으면 분리 대상이다

## 분리 패턴

- **핸들러 함수**: 같은 이름의 폴더를 만들고 함수별 파일로 분리 (예: `tools.ts` → `tools/get-schema.ts`)
- **인터페이스/타입**: `interfaces/` 폴더에 구현체와 같은 파일명으로 분리
- **라우팅/정의**: 원본 파일에 남기고, 구현부만 분리한다
